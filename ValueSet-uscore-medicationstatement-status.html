ValueSet: US Core Medication Status Codes

Official URL: http://www.fhir.org/guides/uscdi4-sandbox/ValueSet/uscore-medicationstatement-status Version: 0.0.0
Standards status: Trial-use Maturity Level: 1 Computable Name: USCoreMedicationstatementStatus

A code specifying the state of the statement. Generally, this will be active or completed state. This value set excludes the concept used internally by the FHIR specification, not-taken,on-hold,stopped, and unknown, because they are covered by the medication adherence extension.

References

  • US Core MedicationStatement

Logical Definition (CLD)

This value set includes codes based on the following rules:

  • Include all codes defined in http://hl7.org/fhir/CodeSystem/medication-statement-status

This value set excludes codes based on the following rules:

  • Exclude these codes as defined in http://hl7.org/fhir/CodeSystem/medication-statement-status
    CodeDisplayDefinition
    not-takenNot TakenThe medication was not consumed by the patient
    on-holdOn HoldActions implied by the statement have been temporarily halted, but are expected to continue later. May also be called 'suspended'.
    stoppedStoppedActions implied by the statement have been permanently halted, before all of them occurred. This should not be used if the statement was entered in error.
    unknownUnknownThe state of the medication use is not currently known.

This value set contains 4 concepts

Expansion based on codesystem Medication status codes v4.0.1 (CodeSystem)

CodeSystemDisplayDefinition
  activehttp://hl7.org/fhir/CodeSystem/medication-statement-statusActive

The medication is still being taken.

  completedhttp://hl7.org/fhir/CodeSystem/medication-statement-statusCompleted

The medication is no longer being taken.

  entered-in-errorhttp://hl7.org/fhir/CodeSystem/medication-statement-statusEntered in Error

Some of the actions that are implied by the medication statement may have occurred. For example, the patient may have taken some of the medication. Clinical decision support systems should take this status into account.

  intendedhttp://hl7.org/fhir/CodeSystem/medication-statement-statusIntended

The medication may be taken at some time in the future.
